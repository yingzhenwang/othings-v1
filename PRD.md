## 1. 产品概述

### 1.1 产品背景

随着个人物品越来越多，很多人难以清晰地了解自己拥有哪些物品、它们的价值、以及何时需要维护或处理。OThings 应运而生，旨在帮助用户系统化管理个人物品。

### 1.2 产品定位

**本地优先的个人物品管理应用**

- 核心价值：让用户轻松掌握自己的所有物品
- 目标用户：个人用户、极客、追求效率的人群
- 差异化：完全本地存储，数据自主可控，无云端依赖

### 1.3 核心功能

| 功能 | 描述 | 优先级 |
|------|------|--------|
| 物品管理 | 添加、编辑、删除、查看物品 | P0 |
| 分类管理 | 创建分类、层级管理 | P0 |
| 智能提醒 | 到期提醒、续费提醒 | P1 |
| 搜索 | 关键词 + 语义搜索 | P1 |
| 报表 | 资产统计、分类分布 | P2 |
| 导入/导出 | 数据备份与恢复 | P2 |

## 2. 用户故事

### 2.1 物品管理

```
作为用户，
我希望能够添加物品的基本信息（名称、数量、分类、位置、状态），
以便清晰地记录我的物品。

作为用户，
我希望能够为物品添加购买日期、价格、保修期等信息，
以便了解物品的价值和保修状态。

作为用户，
我希望能够为物品添加自定义字段，
以满足个性化需求。
```

### 2.2 分类管理

```
作为用户，
我希望能够创建多个分类，并选择颜色和图标，
以便快速识别不同类型的物品。

作为用户，
我希望能够创建二级分类，
以便更精细地组织物品。
```

### 2.3 提醒功能

```
作为用户，
我希望能够在物品保修到期前收到提醒，
以便及时处理或续保。

作为用户，
我希望能够自定义提醒时间（提前N天），
以便根据自己的需求设置提醒。
```

### 2.4 搜索功能

```
作为用户，
我希望能够通过关键词快速找到物品，
以便快速定位。

作为用户，
我希望能够用自然语言描述我的需求并找到相关物品，
以便更智能地搜索。
```

### 2.5 数据报表

```
作为用户，
我希望能够看到我的物品总数和总价值，
以便了解我的资产状况。

作为用户，
我希望能够看到各类物品的数量分布，
以便了解物品的构成。
```

## 3. 功能需求

### 3.1 物品管理

#### 3.1.1 物品列表

- 展示所有物品，支持分页
- 支持网格视图和列表视图切换
- 显示：名称、分类、数量、状态、位置
- 支持排序：名称、创建时间、更新时间、价格
- 支持筛选：分类、状态、位置

#### 3.1.2 添加物品

必填字段：
- 名称 (name)

可选字段：
- 分类 (categoryId)
- 数量 (quantity)，默认 1
- 描述 (description)
- 位置 (location)
- 状态 (status)：active | inactive | discarded，默认 active
- 购买价格 (purchasePrice)
- 购买日期 (purchaseDate)
- 保修到期 (warrantyExpiry)
- 自定义字段 (customFields)

#### 3.1.3 编辑物品

- 可编辑所有字段
- 支持批量编辑

#### 3.1.4 删除物品

- 单个删除
- 批量删除
- 删除前确认

### 3.2 分类管理

#### 3.2.1 分类列表

- 展示所有分类
- 显示：名称、颜色、图标、物品数量
- 支持一级/二级分类

#### 3.2.2 创建分类

必填字段：
- 名称 (name)
- 颜色 (color)

可选字段：
- 图标 (icon)
- 父分类 (parentId)

#### 3.2.3 编辑分类

- 可编辑所有字段

#### 3.2.4 删除分类

- 删除时处理关联物品（转移至"未分类"或一并删除）
- 删除前确认

### 3.3 提醒功能

#### 3.3.1 提醒列表

- 展示所有提醒
- 按状态分组：已过期、即将到期、已完成
- 显示：关联物品、标题、到期日、状态

#### 3.3.2 创建提醒

必填字段：
- 关联物品 (itemId)
- 标题 (title)
- 到期日 (dueDate)

可选字段：
- 提前提醒天数 (notifyBefore)，默认 7 天

#### 3.3.3 提醒状态

- 系统根据当前日期自动判断状态
- 用户可手动标记为已完成
- 已完成提醒可查看完成时间

### 3.4 搜索功能

#### 3.4.1 关键词搜索

- 支持名称、描述、位置搜索
- 支持多关键词组合
- 实时搜索

#### 3.4.2 LLM 语义搜索 (v1.1)

- 支持自然语言描述
- 调用 MiniMax API
- 智能匹配相关物品

### 3.5 报表功能 (v1.1)

#### 3.5.1 资产总览

- 物品总数
- 物品总价值
- 活跃/非活跃物品数量

#### 3.5.2 分类分布

- 各分类物品数量
- 各分类物品价值
- 饼图/柱状图展示

#### 3.5.3 提醒统计

- 即将到期提醒数量
- 已过期提醒数量

### 3.6 导入/导出

#### 3.6.1 导出

- 导出为 JSON 格式
- 包含所有数据：物品、分类、提醒、设置
- 包含导出时间戳和版本号

#### 3.6.2 导入

- 从 JSON 文件导入
- 导入前预览：新增数、冲突数
- 支持选择性导入

### 3.7 设置

- 主题：浅色/深色/跟随系统
- 默认视图：网格/列表
- 每页显示数量
- 搜索模式：普通/智能
- 通知开关

## 4. 非功能需求

### 4.1 性能

- 页面加载时间 < 1s
- 搜索响应时间 < 500ms
- 1000+ 物品无明显卡顿

### 4.2 兼容性

- Chrome >= 90
- Firefox >= 88
- Safari >= 14
- Edge >= 90

### 4.3 数据安全

- 所有数据存储在用户浏览器 localStorage
- 无云端传输，数据完全自主可控
- 导入/导出为唯一的数据备份方式

### 4.4 可用性

- 符合常用交互习惯
- 支持键盘操作
- 表单必填项明确提示
- 错误信息清晰易懂
